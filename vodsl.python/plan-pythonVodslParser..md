# Plan: Single-Source VODSL Python Toolchain

## Context

`net.ivoa.vodsl/src/net/ivoa/vodsl/Vodsl.xtext` is the sole grammar source for both the
Java/Xtext and Python implementations. A Python transpiler script reads it, applies
deterministic rewrites for incompatible constructs, and emits a textX grammar into
`vodsl.python/` as a generated artifact. Hand-written Python layers implement scoping,
validation, and VO-DML generation with schema-validation against the live v1.0 XSD.

## Requirements (settled)

- `Vodsl.xtext` is the single source of truth; future grammar edits need only happen once.
- Generated parser-input artifacts (textX grammar) go in `vodsl.python/src/vodsl/grammar/` and are regenerated by running `generate_grammar.py`.
- Validation rules are ported from `net.ivoa.vodsl/src/net/ivoa/vodsl/validation/VodslValidator.xtend`.
- Output XML must be schema-valid against `https://raw.githubusercontent.com/ivoa/vo-dml/refs/heads/main/xsd/vo-dml-v1.0.xsd`.
- Xtext constructs not directly representable in textX use deterministic rewrite rules (see table below).
- Schema XSD is fetched and cached locally on first use; re-fetched with `--refresh-schema` flag.
- Transpiler is invoked manually (not as part of install hook).

## Steps

### 1  Transpiler script

Write `vodsl.python/generate_grammar.py` that:
- Reads `../net.ivoa.vodsl/src/net/ivoa/vodsl/Vodsl.xtext` (path relative to `vodsl.python/`).
- Parses Xtext rules using a lightweight Python Xtext-grammar reader (regex + line-by-line).
- Applies the six deterministic rewrites catalogued in ¬ß2.
- Writes `vodsl.python/src/vodsl/grammar/vodsl.tx`, stamped with a "do not edit ‚Äì regenerate
  with `generate_grammar.py`" header comment and the source file hash for change detection.

### 2  Six deterministic rewrites

| # | Xtext construct | Location in grammar | Deterministic rewrite |
|---|---|---|---|
| 1 | Unordered group `&` | `Attribute` (line 128‚Äì130), `Composition` (line 136‚Äì139) | Fixed order: `(semanticConcept=SemanticConcept)?  (constraints+=Constraint)*` |
| 2 | Syntactic predicate `=>'.'` | `QualifiedName` rule | Drop predicate; emit rule as textX regex terminal `QNAME:/([A-Za-z_]\w*:)?[A-Za-z_]\w*(\.[A-Za-z_]\w*)*/` |
| 3 | `returns ecore::EString` on `VERSION` | line 32 | Strip annotation; emit as regex terminal `VERSION:/\d+(\.\d+)?/` |
| 4 | `returns ecore::EInt` on `SINT` | line 154 | Strip annotation; emit as regex terminal `SINT:/-?\d+/` |
| 5 | `terminal PI_TEXT: '!' (!'!')* '!'` | line 221 | Emit as textX regex terminal `PI_TEXT:/![^!]*!/` |
| 6 | `enum MultiplicityBound` / `enum ConstraintLanguage` | lines 159, 196 | Emit as textX `enum` blocks verbatim |

Additional mappings that need no rewrite but must be handled:
- Cross-references `[Type|QualifiedName]` ‚Üí captured as plain string in textX; resolved
  post-parse in `scoping.py`.
- `{Constraint}` forced-action ‚Üí dropped (textX always creates an object when a rule matches).
- `abstract?='abstract'` boolean assignment ‚Üí passes through unchanged.
- `ValidID: ID` ‚Üí collapsed to textX built-in `ID`.
- `grammar ‚Ä¶ with org.eclipse.xtext.common.Terminals` header ‚Üí stripped.
- `import "‚Ä¶" as ecore` and `generate vodsl ‚Ä¶` directives ‚Üí stripped.
- Xtext `STRING` ‚Üí textX `STRING` (double-quoted), `ID` ‚Üí textX `ID`, `INT` ‚Üí textX `INT`.

### 3  Parser layer

`vodsl.python/src/vodsl/parser.py`:
- Loads generated `vodsl.tx` via textX `metamodel_from_file`.
- Registers custom textX `obj_processors` to post-process cross-reference strings through
  `scoping.py`.
- Exposes `parse_file(path) -> VoDataModel` and `parse_string(text) -> VoDataModel` APIs.

### 4  Scoping / qualified-name layer

`vodsl.python/src/vodsl/scoping.py` mirrors:
- `VodslQualifiedNameConverter.java`: `to_string(qname_parts)` joins with `:` after first
  segment, `.` thereafter; `to_parts(qname_string)` splits on `[:\.]`.
- `VodslQualifiedNameProvider.java`: root qualified name of a `VoDataModel` is its
  `model.name`.
- `VodslRuntimeModule` import-URI semantics: when resolving a cross-reference, search the
  current model's scope first, then recursively search models brought in via `include`
  declarations.

### 5  Validator layer

`vodsl.python/src/vodsl/validator.py` ports all `@Check` rules from `VodslValidator.xtend`:

| Java check | Python function | Severity |
|---|---|---|
| `checkAttributeType` | `check_attribute_type` | ERROR: "Attribute '{name}' should be a value type" |
| `checkMultiplicity(Attribute)` ‚Äì `ATLEASTONE` | `check_multiplicity_attribute` | ERROR: "multiplicity '+' strongly discouraged for attribute {name}" |
| `checkMultiplicity(Attribute)` ‚Äì `MANY` | same | ERROR: "multiplicity '*' strongly discouraged for attribute {name}" |
| `checkMultiplicity(Attribute)` ‚Äì `ONE` with min‚âÝmax | same | ERROR: "multiplicity for attribute should specify array size so that min=max for {name}" |
| `checkMultiplicity(Reference)` ‚Äì `ATLEASTONE` | `check_multiplicity_reference` | WARNING |
| `checkMultiplicity(Reference)` ‚Äì `MANY` | same | ERROR |
| `checkMultiplicity(Reference)` ‚Äì `ONE` with maxOccurs>1 | same | WARNING |
| `checkMultiplicity(Composition)` ‚Äì max < min (numeric) | `check_multiplicity_composition` | ERROR: "maximum multiplicity less than minimum - {name}" |

Validator returns a list of `Issue(severity, message, element)` objects; errors abort
generation, warnings are printed but generation proceeds.

### 6  Generator layer

`vodsl.python/src/vodsl/generator.py` ports `VodslGenerator.xtend` rule-for-rule:

- **Root `<vo-dml:model>` envelope**: namespace/schema-location attributes, `vodmlVersion="1.1"`,
  `<name>`, `<description>`, `<uri/>`, `<title>`, `<author>` (√óN), `<version>`,
  `<lastModified>` (ISO-8601 UTC from file mtime).
- **`<import>` blocks**: for each `IncludeDeclaration`, resolve the `.vodsl` ‚Üí `.vo-dml.xml`
  filename, parse the included model to extract `name`/`version`, emit `<import>`.
- **Element ordering** (mirrors the `vodml(List<ReferableElement>)` overload):
  primitives ‚Üí enumerations ‚Üí dataTypes ‚Üí objectTypes ‚Üí packages (recursive).
- **`<vodml-id>`**: `fullyQualifiedName.skipFirst(1)` ‚Äî drop the model-name prefix segment,
  join remainder with `.`.
- **`<vodml-ref>`**: full `model:pkg.type` form from `VodslQualifiedNameConverter`.
- **Multiplicity**: `OPTIONAL`‚Üí`[0,1]`, `MANY`‚Üí`[0,-1]`, `ATLEASTONE`‚Üí`[1,-1]`, `ONE`‚Üí`[1,1]`
  unless explicit `@[min,max]` form; explicit `@[n]` ‚Üí `[n,n]`.
- **`NaturalKey` rank**: use explicit `ofRank` if set; otherwise auto-assign 1-based position
  among `iskey` attributes in declaration order.
- **`ProcessingInstruction`**: strip surrounding `!` delimiters, split on first space to get
  `name` + `content`, emit `<?name content?>` (replace `?>` ‚Üí `? >` in content).
- **`SubSet`**: `<constraint xsi:type="vo-dml:SubsettedRole">`.
- **`SemanticConcept`**: `<semanticconcept>` with optional `<topConcept>` and `<vocabularyURI>`.
- **`Constraint`**: `<constraint><description><![CDATA[‚Ä¶]]></description></constraint>`.

### 7  Schema-validation gate

After generation:
- Check for cached XSD at `vodsl.python/src/vodsl/schema/vo-dml-v1.0.xsd`.
- If absent (or `--refresh-schema` passed), fetch
  `https://raw.githubusercontent.com/ivoa/vo-dml/refs/heads/main/xsd/vo-dml-v1.0.xsd`
  and cache it.
- Validate with `lxml.etree` `XMLSchema`; collect and report all errors with line numbers.

### 8  CLI

`vodsl.python/src/vodsl/cli.py` (entry point `vodsl`):

```
vodsl [--output DIR] [--refresh-schema] [--no-validate] <file.vodsl> [<file.vodsl>‚Ä¶]
```

Mirrors `ParserRunner.main`: parse ‚Üí validate ‚Üí generate ‚Üí schema-check, emitting
`<stem>.vo-dml.xml` in the output directory (default: same directory as input).

## Integration test corpus (remote fixtures from ivoa/vodsl-models)

Integration tests fetch two real-world fixtures from the `ivoa/vodsl-models` repository and
cache them under `tests/fixtures/` at test-collection time (a `conftest.py` session-scoped
fixture performs the download, skipping if offline).

### `example.vodsl`

Source: `https://raw.githubusercontent.com/ivoa/vodsl-models/refs/heads/master/example.vodsl`

This file uses `include "IVOA-v1.0.vodsl"`, so the test fixture downloader must also fetch
`https://raw.githubusercontent.com/ivoa/vodsl-models/refs/heads/master/IVOA-v1.0.vodsl`
and place it alongside `example.vodsl` in `tests/fixtures/`.

Expected behaviour:
- Parses without any errors.
- All cross-references resolve correctly (including references into the included `ivoa` namespace,
  e.g. `ivoa:Quantity`, `ivoa:RealQuantity`, `ivoa:string`, `ivoa:anyURI`, `ivoa:integer`,
  `ivoa:boolean`).
- Generates schema-valid VO-DML XML (validated against the cached v1.0 XSD).
- Validator produces **no errors**; the only expected warnings are for `@*`/`@+` attributes in
  the `multiplicities` otype (`m2`, `m3`) ‚Äî these match `checkMultiplicity(Attribute)` WARNING
  level per the xtend rules.

### `test.vodsl`

Source: `https://raw.githubusercontent.com/ivoa/vodsl-models/refs/heads/master/test.vodsl`

This file deliberately exercises every failure mode. Tests must assert that the toolchain
detects exactly the following issues (and does **not** generate XML when errors are present):

#### Parse errors (abort before validation)

| Location | Construct | Reason |
|---|---|---|
| `otype refers { r1 references ; }` | `r1 references ;` | Type reference target removed by block comment ‚Äî missing required type name |
| `@[2 .. 2]` on `m5` | `..` separator | Grammar uses `,` not `..` between min and max in `@[min,max]` |
| Final two `%` characters | Stray `%` at end of file | Not valid VODSL syntax |

#### Scope / resolution errors (detected after parse, before validation)

| Location | Attribute | Reason |
|---|---|---|
| `f4 : daft` | `f4` | `daft` is not declared anywhere in current or included models |
| `f6 : sprim` | `f6` | `sprim` is declared in `topl.p2` but not visible without qualification from `topl.p1` |
| `f8 : obscore:Access` | `f8` | `obscore` namespace is not imported via any `include` declaration |

#### Validation errors (from ported `VodslValidator` checks)

| Location | Check | Issue |
|---|---|---|
| `m2 : ivoa:integer @*` | `check_multiplicity_attribute` ‚Äî MANY | ERROR: "multiplicity '*' strongly discouraged for attribute m2" |
| `m3 : ivoa:integer @+` | `check_multiplicity_attribute` ‚Äî ATLEASTONE | ERROR: "multiplicity '+' strongly discouraged for attribute m3" |
| `m6 : ivoa:integer @[3 .. 2]` | `check_multiplicity_composition/attribute` ‚Äî max < min | ERROR: "maximum multiplicity less than minimum - m6" |
| `r2 @* references osample` | `check_multiplicity_reference` ‚Äî MANY | ERROR: "multiplicity '*' not advised for reference r2" |
| `d3 references dsample` | `check_attribute_type` / reference target type check | ERROR: reference target `dsample` is a `DataType`, not an `ObjectType` |
| `dtype badderivation2 -> osample` | supertype type check | ERROR: `osample` is an `ObjectType`, not a valid `DataType` supertype |

#### Validation warnings

| Location | Check | Issue |
|---|---|---|
| `r3 @+ references osample` | `check_multiplicity_reference` ‚Äî ATLEASTONE | WARNING: "multiplicity '+' not advised for reference r3" |
| `r1 @? references osample` | `check_multiplicity_reference` ‚Äî OPTIONAL | No issue (OPTIONAL is explicitly allowed for references) |

#### Generation behaviour

Because `test.vodsl` contains errors, `generator.py` **must not** produce output XML. The
test asserts that `cli.py` exits with a non-zero status and that no `.vo-dml.xml` file is
written to the output directory.
